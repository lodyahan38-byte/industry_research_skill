# Industry Research OS Skill

这是一个用于 **行业调研、赛道分析、竞品研究、机会判断、持续行业情报追踪** 的 ChatGPT Skill。

它的核心不是让 AI 直接写一篇泛泛的行业报告，而是让 AI 按照以下流程工作：

```text
明确调研目标
↓
定义行业边界
↓
按 P0 / P1 / P2 模块推进
↓
每完成一个模块都做 Gate Review
↓
判断是否进入下一步、修补、扩展信息源、追问用户或停止下结论
```

## 文件说明

```text
.
├── SKILL.md      # 行业调研 Skill 主体
└── README.md     # 使用说明与 Scheduled Tasks 配置指南
```

## Skill 适用场景

适合以下任务：

- 行业调研
- 市场前景研究
- 赛道分析
- 竞品分析
- 产品机会判断
- 商业模式研究
- 求职行业判断
- 招聘市场 / 岗位趋势追踪
- 内容选题与关键词研究
- 每周行业情报更新

## 核心机制：Gate Review

每个模块结束后，AI 必须输出质量审查结果：

| Gate Result | 含义 | 下一步 |
|---|---|---|
| PASS | 当前模块合格 | 进入下一模块 |
| REPAIR | 当前模块结构有了，但质量不够 | 修补当前模块 |
| EXPAND | 信息源、样本或数据明显不足 | 继续补充信息源 |
| ASK_USER | 缺少用户目标、地区、时间、对象等关键约束 | 最多追问 1-2 个问题 |
| STOP | 当前信息不足以支撑继续判断 | 停止下结论，说明原因 |

## 推荐使用方式

### 方式 1：直接复制 SKILL.md

把 `SKILL.md` 的内容复制进 ChatGPT 对话，然后输入：

```text
我想研究【行业名称】，目标是【求职 / 产品规划 / 竞品分析 / 市场前景研究】。
请使用 Industry Research OS Skill，从 P0 开始推进，并在每一步输出 Gate Review。
```

### 方式 2：配合 ChatGPT Scheduled Tasks 做定时任务

ChatGPT Scheduled Tasks 适合做：

- 每日招聘信息简报；
- 每周求职市场复盘；
- 每周行业情报更新；
- 每月行业地图更新；
- 定期提醒你更新判断日志。

注意：Scheduled Tasks 目前适合做定时检查和定时输出，不适合复杂的事件触发自动化；它也不能直接运行 GPTs。为了稳定，建议把下面的任务 prompt 直接粘到 Scheduled Task 里。

OpenAI 官方说明入口：
https://help.openai.com/en/articles/10291617-scheduled-tasks-in-chatgpt

# Recruiting Scheduled Tasks 配置指南

这里的 recruiting 指的是：**AI 产品经理 / AI 产品运营 / AIGC 产品相关岗位的招聘市场追踪与求职推进**。

## 1. 推荐任务结构

不要只建一个很大的任务。建议拆成 3 个任务：

| 任务 | 频率 | 目标 |
|---|---|---|
| Daily Recruiting Brief | 工作日每天早上 | 看今天有哪些值得关注的岗位和投递动作 |
| Weekly Recruiting Review | 每周一次 | 复盘一周招聘市场、投递进展和能力缺口 |
| Monthly Market Map Update | 每月一次 | 更新 AI 产品岗位市场地图和公司优先级 |

如果只想先跑通，先建第一个：`Daily Recruiting Brief`。

---

## 2. 在 ChatGPT 里创建 Scheduled Task

### 操作步骤

1. 打开 ChatGPT 网页端或移动端。
2. 在侧边栏找到 `Scheduled` / `Scheduled Tasks` 页面。
3. 点击新建任务。
4. 选择任务运行时间，例如：
   - 工作日每天 08:30；
   - 每周一 09:00；
   - 每周日 20:30。
5. 把下面的 prompt 粘进去。
6. 开启通知。
7. 第一次运行后，检查输出是否符合你的求职目标；如果不符合，编辑任务 prompt。

## 3. Daily Recruiting Brief Prompt

适合设置为：**工作日每天早上 08:30**。

```text
请每天帮我做 AI 产品方向的 Recruiting Brief。

我的求职目标：
- 目标岗位：AI 产品经理、AI 产品运营、AIGC 产品、AI 应用产品、AI 工具产品相关实习 / 校招岗位；
- 优先方向：AI 应用、AIGC 视频、AI Agent、法律 AI / 司法 AI、AI 工作流、内容生产工具；
- 关注城市：深圳、北京、杭州、上海、广州，优先深圳南山区、北京海淀区、杭州；
- 岗位阶段：实习、暑期实习、校招提前批、转正机会；
- 我的背景关键词：AI 产品实习、AIGC 视频工作流、prompt 工程、badcase 评测、飞书 / Codex / Coze 工作流、竞品分析、需求分析、PRD / SOP 文档沉淀。

请你执行以下任务：

## 1. 今日岗位信号
请查找或整理今天值得关注的 AI 产品相关岗位。
如果你无法直接访问招聘网站，请明确说明，并给我一组今天应该手动搜索的关键词。

输出字段：
岗位名称｜公司｜城市｜岗位方向｜为什么匹配我｜需要补的能力｜优先级｜下一步动作

## 2. 招聘市场变化
请判断最近 AI 产品岗位更看重哪些能力：
- 产品设计；
- AI 工具使用；
- 数据分析；
- Prompt / Agent 工作流；
- 多模态 / AIGC；
- 文档能力；
- 商业分析；
- 行业理解。

## 3. 今日投递建议
请给我今天最小可执行行动，只给 1-3 个：
- 应该投哪些方向；
- 应该改简历哪一段；
- 应该补哪个作品集描述；
- 应该准备哪个面试问题。

## 4. 风险提醒
请指出我今天最容易陷入的误区，例如：
- 只收藏岗位不投递；
- 只焦虑能力缺口；
- 简历没有针对 JD 改；
- 把 AI 产品、AI 运营、产品设计混成一类。

## 5. Recruiting Gate Review
请对今天的简报做质量审查：
| 维度 | 分数 | 原因 |
|---|---:|---|
| 岗位相关性 | /5 |  |
| 信息新鲜度 | /5 |  |
| 与我背景匹配度 | /5 |  |
| 行动可执行性 | /5 |  |
| 风险识别 | /5 |  |

最后输出：
- 今日最重要判断：
- 今日最小下一步：
- Gate Result：PASS / REPAIR / EXPAND / ASK_USER / STOP
```

## 4. Weekly Recruiting Review Prompt

适合设置为：**每周日晚上 20:30** 或 **每周一上午 09:00**。

```text
请帮我做本周 AI 产品求职复盘和下周 recruiting 计划。

请围绕 AI 产品经理 / AI 产品运营 / AIGC 产品 / AI Agent 产品岗位，输出以下内容：

## 1. 本周招聘市场变化
- 哪些公司 / 方向更活跃？
- 岗位要求出现了哪些高频能力？
- 哪些能力要求正在变强？
- 哪些岗位看起来不适合我？

## 2. 本周岗位分类
请把本周值得关注的岗位分成：
1. 高匹配：可以优先投；
2. 中匹配：需要改简历后投；
3. 低匹配：暂时观察；
4. 不建议投：原因明确。

输出表格：
岗位方向｜代表公司｜匹配原因｜缺口｜优先级｜下周动作

## 3. 我的能力缺口复盘
请从以下维度判断我下周最应该补什么：
- 产品设计 / 原型；
- 数据分析 / SQL；
- AI Agent / 工作流；
- AIGC 视频 / 多模态；
- 竞品分析；
- 行业研究；
- 面试表达；
- 作品集表达。

## 4. 下周投递策略
请给我一个非常具体的下周计划：
- 下周优先投递的 3 类岗位；
- 下周必须优化的 1 个简历模块；
- 下周必须准备的 3 个面试问题；
- 下周最小投递目标。

## 5. 判断日志
请记录本周新增判断：
| 判断内容 | 证据 | 置信度 | 反对证据 | 下周验证方式 |
|---|---|---|---|---|

## 6. Gate Review
请评价本周复盘质量：
| 维度 | 分数 | 原因 |
|---|---:|---|
| 市场观察 | /5 |  |
| 岗位分类 | /5 |  |
| 能力缺口判断 | /5 |  |
| 下周计划可执行性 | /5 |  |
| 判断日志质量 | /5 |  |

最后输出：
- 本周最重要判断：
- 下周最小下一步：
- Gate Result：PASS / REPAIR / EXPAND / ASK_USER / STOP
```

## 6. 使用注意事项

### 1. Scheduled Tasks 不等于完整自动化系统

它适合定时提醒、定时搜索、定时总结和定时检查，但不等于完整的飞书 / Notion / 招聘网站自动写入系统。

### 2. 不要依赖它直接替你完成投递

Recruiting 任务的目标是：

```text
每天减少启动成本
↓
帮你识别优先级
↓
推动你完成最小下一步
```

不是让 AI 完全替你找工作。

### 3. 每次输出都必须有 Gate Review

如果任务只给你一堆岗位，但没有判断质量、优先级和下一步动作，就要回去改 prompt。

### 4. 遇到信息不足，要让它说清楚

任务必须允许 AI 输出：

```text
当前无法确认最新招聘信息，需要你补充招聘平台链接 / JD 截图 / 目标公司清单。
```

不能让它在没有来源的情况下编岗位。
